    # 插件(plugins)

    插件是 webpack 的支柱功能。 webpack 本身就是一个插件 他是由很多插件组合而成的 ，在打包的时候，在不同的位置上 执行对应的插件。在配置中配置 plugins，值为 Array。

    前言：所有配置区分开发与生产环境。基本配置；目录结构如下：

    ├── build
│   ├── webpack.base.js       //基础，共用配置
│   ├── webpack.dev.js        //开发环境的配置
│   └── webpack.prod.js       //生产环境的配置

    # 一、webpack-dev-server （开发使用）

    1. 作用： 用来快速搭建本地运行环境的工具，即配置开发服务器。
    2. 安装： npm install webpack-dev-server --save-dev 3. 使用：
    3.1 配置命令（一般配置为dev，当然还可以配置为其他的命令）

    • 基本使用 默认开启 8081 端口，通过http://localhost:8081/访问
    "scripts": {
    "dev": "webpack-dev-server"
  },
    • 搭配webpack使用（常用），将打包配置做为参数传给 webpack-dev-server
    "scripts": {
    "dev": "webpack-dev-server --config ./build/webpack.base --env.development"
  },

    参数解析：
    --config ./build/webpack.base：以./build/webpack.base.js为打包配置文件；
    --env.development：打包mode为development，即打包开发环境

    • 运行
    npm run dev

    4. 自定义配置webpack-dev-server
    注意：该插件为开发配置，所以这个配置在webpack.dev.js中，可以不用配置。就使用它默认的也是可以的。
    常用配置如下：详细配置

    const path = require('path');
module.exports = {
    mode: 'development',     // 模式配置 'development'
    devServer: {
        // contentBase告诉服务器从哪里提供内容。这仅在您希望提供静态文件时才有必要。devServer。publicPath将用于确定服务包的位置
        contentBase: path.resolve(__dirname, '../dist'),          
        compress: true,             //启用gzip压缩，node服务压缩的一种
        port: 9000,                 //修改端口号为9000
        host: '127.0.0.1',          //服务器的主机号，默认是localhost，修改后为 http://127.0.0.1:9000/
        hot: true,                  //启动热更新
        open: true,                 //在服务器启动后打开默认浏览器
        proxy: {
            '/api': 'http://localhost:3000'         //设置代理，解决跨域
        },
        //publicPath用于设置编译后文件的路径，假设服务器的运行地址是 http://localhost:8080，输出文件名设置为bundle.js，那么默认情况下publicPath是”/”，因此文件地址为http://localhost:8080/bundle.js
        publicPath: '/assets/',     //设置后文件地址为：http://localhost:8080/assets/bundle.js
        overlay: true,              //编译错误或警告时，在浏览器中显示全屏覆盖，默认为false
  
        inline: false,              //启动 inline mode 模式自动刷新，默认是 Iframe mode 模式
    }
}

    属性 inline 的作用： iframe与inline的区别

    • Iframe mode 是在网页中嵌入了一个 iframe，将我们自己的应用注入到这个 iframe 当中去，因此每次你修改的文件后，都是这个 iframe 进行了 reload。配置时不需要添加其他的内容 ==>> 浏览器访问的路径是:localhost:8080/webpack-dev-server/index.html
    • Inline-mode，是 webpack-dev-server 会在你的 webpack.config.js 的入口配置文件中再添加一个入口。这意味着将在包中插入一个脚本来处理实时的重新加载，构建消息将出现在浏览器控制台中。配置是需要配置 inline 属性，浏览器访问的路径是:localhost:8080/index.html
