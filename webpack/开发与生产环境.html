# 前言：实际项目中，我们都会根据开发环境与生产环境不同，配置不同的webpack（如代码压缩）；打包时提供mode属性来区分是开发环境还是生产环境，来实现配置文件的拆分

# 1. 当前目录下新建配置文件夹 build，并创建 基础、开发、生产环境的配置文件

├── build
│   ├── webpack.base.js       //基础，共用配置
│   ├── webpack.dev.js        //开发环境的配置
│   └── webpack.prod.js       //生产环境的配置

# 2. package.json 配置打包命令

"scripts": {
    "build": "webpack --config ./build/webpack.base --env.production",
    "dev": "webpack --config ./build/webpack.base --env.development"
},

# 3. 合并配置文件

判断当前环境是否是开发环境来加载不同的配置，使用webpack-merge做配置合并 安装：

npm install webpack-merge --save-dev

webpack.dev配置

module.exports = {
    mode:'development'
}

webpack.prod配置

module.exports = {
    mode:'production'
}

webpack.base配置

const path = require('path');
const merge = require('webpack-merge');
// 开发环境
const dev = require('./webpack.dev');
// 生产环境
const prod = require('./webpack.prod');
const base = { // 基础配置
    entry:'./src/index.js',
    output:{
        filename:'bundle.js',
        path:path.resolve(__dirname,'../dist')
    }
}
module.exports = (env) =>{
    if(env.development){      //如果是运行的是 `npm run dev` ==> 开发环境
        return merge(base,dev);     //合并base和dev的配置
    }else{                    //如果是运行的是 `npm run build` ==> 生产环境
        return merge(base,prod)     //合并base和prod的配置
    }
}
