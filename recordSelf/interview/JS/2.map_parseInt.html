    # ['1', '2', '3'].map(parseInt) what & why ?

    # 1. 首先，看一下执行结果

    console.log(['1', '2', '3'].map(parseInt));   //[1, NaN, NaN]

    # 2. 知识点解析

    # 2.1 parseInt

    parseInt() 函数解析一个字符串参数，并返回一个指定基数的整数。返回一个整数或NaN。
    即将某种进制（10进制、8进制、16进制）的字符串 转为10进制的数字！

        const value = parseInt(string, radix)
    参数 描述
    string 必需。要被解析的字符串。如果参数不是一个字符串，则将其转换为字符串(类似 toString 的操作)。字符串开头的空白符将会被忽略。
    radix 可选。表示要解析的数字的基数。该值介于 2 ~ 36 之间的整数。 默认为10。即表示string是什么进制
    parseInt('F', 16);    //15 => 指定F为16进制
parseInt('0xF');      //15 => 以"0x"开头，默认字符串为16进制
parseInt('17', 8);    //15 => 指定字符串为8进制
parseInt('015');      //15 => 以"0"开头，字符串可能为10进制，但已有可能为8进制，结果为13

    注意：在radix为 undefined，或者radix为 0 或者没有指定的情况下，JavaScript 作如下处理：

    • 如果字符串 string 以"0x"或者"0X"开头, 则基数是16 (16进制).
    • 如果字符串 string 以"0"开头, 基数是8（八进制）或者10（十进制），那么具体是哪个基数由实现环境决定。ECMAScript 5 规定使用10，但是并不是所有的浏览器都遵循这个规定。因此，永远都要明确给出radix参数的值。
    • 如果字符串 string 以其它任何值开头，则基数是10 (十进制)。
    • 如果省略该参数或其值为 0，则数字将以 10（十进制） 为基础来解析。
    • 如果该参数小于 2 或者大于 36，则 parseInt() 将返回 NaN。

    更多详情见 parseInt | MDN

    # 2.2 map

    map() 方法创建一个新数组，其结果是该数组中的每个元素都调用一个提供的函数后返回的结果，返回值是一个新的数组。

    var new_array = arr.map（function callback（currentValue [，index [，array]]）{
     //返回new_array的元素
} [，thisArg ]）
    参数 描述
    callback 必须。函数，数组中的每个元素都会执行这个回调函数函数
    thisArg 可选, 执行 callback 函数时使用的this 值。如果省略了 thisValue，或者传入 null、undefined，那么回调函数的 this 为全局对象。
    callback参数 描述
    currentValue 必须。当前元素的值
    index 可选。当前元素的索引值。
    array 可选。当前元素属于的数组对象。即数组本身

    常见用法：

    const arr = [1, 2, 3];
arr.map((num) => num + 1); // [2, 3, 4]

    更多详情见 Array.prototype.map（） | MDN

    # 3. 题目解析

    ['1', '2', '3'].map(parseInt)   // 即数组被map遍历，map的回调函数是parseInt()

    分析：

    • 首先，map()的回调函数默认会有三个参数，即 item,index,arr，并且回调函数会有返回值
    • 但是，parseInt最多只能接受两个参数，即要解析的字符串和字符串对应的奇数

    所以，实际执行的的代码是：

    ['1', '2', '3'].map((item, index) => {
	return parseInt(item, index)
})

    相当于

    parseInt('1', 0)    // 1，radix（基数为0），代表字符串为10进制，
parseInt('2', 1)    // NaN，radix 小于2，返回NaN
parseInt('3', 2)    // NaN，radix为2 代表 3 为二进制，但二进制中没有 3，所以也返回了NaN

    所以，题目执行结果为：

    ['1', '2', '3'].map(parseInt)   //[1, NaN, NaN]

    同理，我们可以看出下面这道题的执行过程：

    ['10','10','10','10','10'].map(parseInt);       // [10, NaN, 2, 3, 4]
