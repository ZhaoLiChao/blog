    # html页面转pdf下载实现

    # 1. 前言

    这段时间在项目开发中，遇到这样一个需求：前端在一个页面中使用echarts生成了几个图表，需要可以将他们导出为一个pdf。虽然echarts自身有下载，但只能 单个图表下载为图片，所以不满需求。而图表又是根据后台给出的数据由前端生成的，所以让后台提供下载的接口也比较麻烦。所以最后有前端生成pdf并下载！

    根据调研，决定采用 html2canvas(文档) 和 jspdf(文档) 的方式实现，即使用 html2canvas 将页面转成图片后，放到pdf中导出。

    # 2. 安装插件

    npm install html2canvas --save-dev;
npm install jspdf --save-dev

    # 3. 核心代码

    import html2canvas from 'html2canvas';
import jsPDF from 'jspdf';

html2canvas(document.getElementById('mainBox'),{            //mainBox 为要导出的html盒子id
   dpi: 200,//导出pdf清晰度
   useCORS: true // 【重要】开启跨域配置
   }).then(function(canvas) {
        // document.body.appendChild(canvas);
        let contentWidth = canvas.width;
        let contentHeight = canvas.height;
 
        //一页pdf显示html页面生成的canvas高度;
        let pageHeight = contentWidth / 592.28 * 841.89;
        //未生成pdf的html页面高度
        let leftHeight = contentHeight;
        //pdf页面偏移
        let position = 0;
        //html页面生成的canvas在pdf中图片的宽高（a4纸的尺寸[595.28,841.89]）
        let imgWidth = 595.28;
        let imgHeight = 592.28 / contentWidth * contentHeight;
 
        let pageData = canvas.toDataURL('image/jpeg', 1);
        let pdf = new jsPDF('', 'pt', 'a4');
 
        //有两个高度需要区分，一个是html页面的实际高度，和生成pdf的页面高度(841.89)
        //当内容未超过pdf一页显示的范围，无需分页
        if (leftHeight < pageHeight) {
            pdf.addImage(pageData, 'JPEG', 0, 0, imgWidth, imgHeight);
        } else {
            while (leftHeight > 0) {
                pdf.addImage(pageData, 'JPEG', 0, position, imgWidth, imgHeight)
                leftHeight -= pageHeight;
                position -= 841.89;
                //避免添加空白页
                if (leftHeight > 0) {
                  pdf.addPage();
                }
            }
        }
        pdf.save('TestReport.pdf');     //导出名为 TestReport.pdf
   });

    # 4. 代码解析

    • 注意： html2canvas 可以单独使用，生成屏幕截图哦！

    • html2canvas 基本使用

    html2canvas(document.body).then(function(canvas) {
    document.body.appendChild(canvas);
});

    • html2canvas 主要有两个参数，即 html2canvas(element, options)
      element : 是要 html2canvas 截图的DOM元素，可以为整个body或者某一个区域（盒子），根据js选择器或者 ref 选中传入即可
      options : 是 html2canvas 的截图参数，请参考 全部参数

    • html2canvas 返回是的一个promise，可以使用 then 获取截图后的结果。

    • 当然，还可以通过在 options 配置 onrendered 属性来获取回调结果，如：

    html2canvas(document.getElementById('mainBox'), {
    allowTaint: true,
    taintTest: false,
    onrendered: function(canvas) {
        var img = canvas.toDataURL('image/jpeg', 1.0);
        //打印出来之后:data:image/jpeg;base64,/9j/4AAQSkZJRg....
        //可以通过chrome来查看
        console.log(img);
    }
});

    # 5. 我踩过的坑

    1. html2canvas 截图只能接到当前屏幕的，滚动到屏幕以外的截取不到？
    解决方法：请注意如果内容高度超过了屏幕高度，请确认是否有滚动条，我就是因为项目对滚动条做了特殊处理，不会显示出来，滚动是自己封装实现的，导致了截图不全的问题。

    还有一种情况，我将下载按钮放在要导出的盒子下方，会导致截图不全，但我将按钮放在盒子上方就可以，没找到原因....（有可能是浏览器渲染机制导致的）

    html2canvas 和 jspdf 这个方法网上有很多现实文章，可自行搜索参考。特别是 html2canvas 的用法，有坑，使用时请注意！
