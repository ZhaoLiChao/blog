    # Vue的基本使用

    # 一、v-model的原理

    v-model 的主要是作用于input输入值的绑定，那么他的原理是什么呢？

    # 1. 通过v-bind和事件实现

    <!-- $event指代的是事件源 -->
<input type="text" :value="value" @input="fn($event)">

    methods:{
    fn(e){
        this.value = e.target.value
    }
},

    # 2. 写的简单一点

    <!-- 只是把复制操作直接写在了事件里面，逻辑并没有变 -->
<input type="text" :value="value" @input="e=>value=e.target.value">

    # 3. v-modal（就是由:value 和 @input实现的）

    <!-- v-model 他就是上面的简写 语法糖 -->
<input type="text" v-model="value">
{{value}}

    # 二、v-html有什么缺点？

    <!-- tmp:'<h1>Hello world</h1>', -->
<span v-html="tmp"></span>

    注意：

    • 不要将用户输入的内容显示到页面上，有可能导致xss攻击；如用户在input输入js代码，获取cookie，调用alert等，而v-html运行了它，就有可能导致xss攻击
    • 后端返回的数据 可以通过v-html来格式化

    # 三、$nextTick 是什么

    # 1. 代码看下（通过实例调用）

    <!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <meta http-equiv="X-UA-Compatible" content="ie=edge">
    <title>Document</title>
</head>
<body>
    <div id="app">{{name}}</div>
    <script src="https://cdn.bootcss.com/vue/2.6.11/vue.min.js"></script>
    <script>
        // vm => 就是 view model（即controller）
        let vm = new Vue({
            el: '#app',         // 内部调用的是querySelector
            data () {
                // 这里就是model层（数据）
                return{
                    name: '张三'
                }
            }
        })
        console.log('======vm.$el', vm.$el)         // id为app的真是DOM
        /**
         * vue有个特点 不会在本轮代码执行的时候 去重新渲染dom
         * 而将在下一个事件环中执行 (promie.then mutationobserver setimmediate settimeout)
        */
       // 所以vm.$nextTick 的回调函数是不会执行的，会在下一次打印
        vm.$nextTick(()=>{          
            console.log('======更新后的DOM', vm.$el); // 渲染后的真实dom
        });
        console.log('======vm.$data', vm.$data)     // vm上的数据对象
        console.log('======vm.$options', vm.$options)   // 整个vm对象
        vm.name = '李四'         // 这里改变了数据，按理来说页面将显示 "李四"
        debugger                // 这里断点，打印出来的真实DOM $el 是 "张三"，在$nextTick里面打印的才是 "李四"
    </script>
</body>
</html>
