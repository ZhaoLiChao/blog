    # new的原理实现

    # 一、new 是什么？

    # 1. 首先，看一段代码，我们用两种方法来创建一个对象（通过构造函数）

    function Person(age){
    this.age= age;
    console.log(this);   // 看看this指向谁
    return {age:age}; // 返回一个对象
}
var person1 = new Person(20);       // 通过new实例化对象
var person2 = Person(18);           // 直接调用构造函数返回一个对象
 
console.log(person1);
console.log(person2);

    # 2. 打印结果：

    两种实例化对象方式，一个使用了new关键字，一个没有。
    从图里可以看出 两者最大的区别在于执行时的执行上下文不同。一个在Person对象中执行，一个在window。前者是真正实例化了一个对象，后者并没有。

    # 二、new，它到底做了什么？

    • 创建一个新对象
    • 将函数的作用域赋给新对象（这里实际上就是生产了一个新的上下文）
    • 执行函数中的代码（为新对象添加属性、方法）
    • 考察第3步返回的返回值，无返回值或者返回一个非对象值时，则将创建的新对象返回，否则会将返回值作为新对象返回
    function myNew() {
    let newObj = {}  // 1. 创建一个新的对象
    let [constructor, ...args] = [...arguments]  // 2. 从所有参数中获取到构造函数和参数列表

    constructor.call(newObj, ...args)   // 3. 通过call改变this指向并执行函数。即通过newObj调用一下构造函数并传入所有参数
    newObj.__proto__ = constructor.prototype    // 4. 强制将新对象的隐式原型指向构造函数的prototype（改变原型链）
    return newObj       // 返回新创建的对象
}

// ================ 测试代码 ================
function Animal(name) {
    this.name= name
}
Animal.prototype.say = function() {
    console.log(this.name + '正在说话')
}
let panda = myNew(Animal, '熊猫')
console.log('=====panda', panda, panda.name)
panda.say()
